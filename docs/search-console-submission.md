# robots.txt + sitemap.xml Review & Search Console Submission

> One page to (a) confirm robots/sitemap are correct, and (b) submit DevStash to
> Google Search Console and Bing Webmaster Tools. Pairs with
> `docs/launch-checklist.md` and `docs/seo-checklist.md`.

---

## 1. robots.txt — review

Generated by `app/robots.ts` (served at `/robots.txt`). Current output:

```
User-agent: *
Allow: /
Disallow: /api/
Disallow: /_next/
Disallow: /admin/
Sitemap: https://devstash.me/sitemap.xml
```

**Verdict: correct for production.**

- ✅ Allows all crawlers on all public content.
- ✅ Disallows `/api/` (no value to crawl the contact/track/og endpoints
  directly), `/_next/` (build internals), and `/admin/` (defense-in-depth; the
  admin is also `noindex` and returns 404 in production).
- ✅ Declares the sitemap, env-aware (`NEXT_PUBLIC_SITE_URL`, falls back to
  `https://devstash.me`).
- ⚠️ The `/api/og` endpoint is under the `/api/` disallow. That's fine — pages
  reference the OG image via their `og:image` meta tag, and social crawlers
  (Facebook, X, LinkedIn) fetch the image URL directly regardless of robots, so
  link previews still work.

**Before submitting:** confirm `https://devstash.me/robots.txt` resolves and the
`Sitemap:` line shows the production domain (it will, once `NEXT_PUBLIC_SITE_URL`
is set in Vercel).

## 2. sitemap.xml — review

Generated by `app/sitemap.ts` (served at `/sitemap.xml`).

**Coverage (verified):**

- Static: `/` (priority 1.0), `/about`, `/projects`, `/blog`, `/resources`,
  `/tools`, `/contact`, `/privacy`, `/terms`.
- Dynamic: every published blog post (`/blog/<slug>`, `lastModified` from
  `post.updatedAt`) and every project (`/projects/<slug>`).

**Verdict: correct.**

- ✅ All public, indexable routes are present; every entry has `lastModified`.
- ✅ **Drafts are excluded** — `getAllPosts()` filters `draft: true`, so unpublished
  posts won't be submitted.
- ✅ `/preview` is **gone** (deleted) and was never in the sitemap.
- ✅ `/admin/*` correctly absent.
- ✅ Blog **category/tag** index pages are intentionally excluded to avoid
  thin/duplicate content competing with the canonical posts. _(If you later want
  them indexed, add them in `app/sitemap.ts`.)_
- ℹ️ `lastModified` for static routes uses build time (`new Date()`), so it
  refreshes on each deploy. Fine; for stricter accuracy you could hardcode dates
  for legal pages, but it's not necessary.

**Before submitting:** load `https://devstash.me/sitemap.xml`, confirm it's valid
XML, the URLs use `https://devstash.me` (not localhost), and the counts look
right (9 static + N posts + M projects).

## 3. Pre-submission gate (do first)

- [ ] Production env set in Vercel: `NEXT_PUBLIC_SITE_URL=https://devstash.me`,
      `GOOGLE_SITE_VERIFICATION`, `BING_SITE_VERIFICATION` (+ analytics + Resend).
- [ ] Site deployed and `https://devstash.me` loads over HTTPS (custom domain
      live; Cloudflare DNS grey-cloud).
- [ ] `https://devstash.me/robots.txt` resolves with the block above.
- [ ] `https://devstash.me/sitemap.xml` resolves as valid XML with production URLs.
- [ ] Spot-check a page's `<head>` for the `google-site-verification` meta tag
      (renders only when `GOOGLE_SITE_VERIFICATION` is set).

## 4. Google Search Console — submission

> The Google verification `<meta>` tag is already wired in `app/layout.tsx` and
> renders when `GOOGLE_SITE_VERIFICATION` is set. So the flow is: get token →
> set env → redeploy → verify → submit sitemap.

1. [ ] Go to https://search.google.com/search-console.
2. [ ] **Add property → URL prefix** → `https://devstash.me`.
3. [ ] Choose **HTML tag** verification; copy the `content="..."` token.
4. [ ] Set `GOOGLE_SITE_VERIFICATION=<token>` in Vercel env → **redeploy**.
5. [ ] Confirm the tag is live (View Source → `<meta name="google-site-verification">`).
6. [ ] Click **Verify** in GSC.
7. [ ] **Sitemaps** → enter `sitemap.xml` → **Submit**. Wait for status
       **Success** (can take minutes to hours).
8. [ ] **URL Inspection** → paste `https://devstash.me` → **Request indexing**
       (do the same for 2–3 key pages: home, top blog post, projects).
9. [ ] After a few days: check **Pages / Coverage** — confirm important pages are
       indexed and nothing important is "Excluded". Check **Core Web Vitals** and
       **Mobile Usability** once data appears.

---

## 5. Bing Webmaster Tools — submission

> The Bing verification meta tag (`msvalidate.01`) is also wired in
> `app/layout.tsx`, gated on `BING_SITE_VERIFICATION`.

1. [ ] Go to https://www.bing.com/webmasters.
2. [ ] **Fastest:** **Import from Google Search Console** (carries over
       verification + sitemap). OR:
3. [ ] **Add site** `https://devstash.me` → **Meta tag** verification → set
       `BING_SITE_VERIFICATION=<token>` in Vercel → redeploy → **Verify**.
4. [ ] **Sitemaps** → submit `https://devstash.me/sitemap.xml`.
5. [ ] Confirm the sitemap is accepted; review **Site Explorer / URL Inspection**
       for crawl issues.

---

## 6. After submission — recurring checks

- [ ] **On each new blog post:** it auto-appears in the sitemap on deploy. For
      faster indexing, run a URL Inspection → Request Indexing for the new URL
      (or wire the Phase 7 `scripts/indexing/` ping later).
- [ ] **Monthly:** GSC Coverage + Core Web Vitals; Bing crawl errors; run
      `pnpm check:links` to catch 404s; confirm sitemap still valid.
- [ ] **When you change a URL/slug:** add a redirect (Next `redirects()` in
      `next.config.ts`) so you don't lose link equity, and update internal links.
